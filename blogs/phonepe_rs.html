My Experience at PhonePe.

Reconciliation Systems


The "Reconciliation Service" project was critical to the daily operations of the company. A reconciliation is a process that takes place on a daily basis where each service provider sends a file to Phonepe containing the details of all the transactions that were recorded on their end. This data is then matched with the transactions that happened on our end to make sure that pending transactions are updated.


The Reconciliation Service project was divided into two types of setups for each Service Provider - API based and SFTP based. In an API based recon, the statuses of all the pending transactions on our end were checked on the service provider's end via API calls, whereas in SFTP based recon the Service provider would upload a file onto an SFTP server which would then be processed by our system in place.


I was responsible for designing both SFTP and API based recon systems for Service providers like Visa CyberSource, Airtel, PayU, and TPSL. The SFTP based recon was built using python and API based recon was built using Java microservices which were written using dropwizard framework. These recon systems cross-verified data for millions of transactions on both ends and sent mailing alerts to the stakeholders in case of discrepancy. The scale of this project was huge and it was a great learning experience. It also helped me to understand the importance of data reconciliation in the day-to-day operations of a company.


In conclusion, working on the "Reconciliation Service" project at Phonepe was a challenging but rewarding experience. It allowed me to develop my skills in building robust systems using python and Java with Dropwizard framework that can handle large volumes of data and also helped me understand the importance of data reconciliation in the day-to-day operations of a company.

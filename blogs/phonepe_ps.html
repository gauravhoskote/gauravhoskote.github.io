My Experience at PhonePe.

Product Service


As a software developer at Phonepe, I had the opportunity to work on a project that aimed to minimize the manual workload on the Operations team - the "Product Service" project. The Operations team was facing a lot of customer requests and complaints that were repeating in nature and required the same standard operating procedure to be followed. These repetitive tasks were taking up a lot of time and resources, and there was a need for a solution that could automate these tasks and reduce the workload.


That's where the "Product Service" project came in. The project involved building a set of APIs written in Java (using the Dropwizard framework) that would automate these repetitive tasks. The APIs were designed to handle customer requests and complaints in a standard and consistent manner, which not only reduced the workload on the Operations team but also improved the overall customer experience.


The project was a great success and it was well-received by the Operations team. It not only helped to automate repetitive tasks but also allowed the team to focus on more important and value-adding tasks. As a result, the Operations team was able to handle customer requests and complaints more efficiently and effectively.


In conclusion, working on the "Product Service" project at Phonepe was a great opportunity for me to develop my skills in building robust systems using Java and the Dropwizard framework.
